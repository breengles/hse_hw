\section{Task 4}
\begin{task}
    Дана обычная бинарная куча (с минимумом в голове), требуется
    узнать $k$-й минимум.
    \begin{enumerate}
      \item $\O(k\log n)$
      \item $\O(k\log k)$
    \end{enumerate}
\end{task}

\begin{solution}
    \begin{enumerate}[a)]
        \item прогоним $k$ раз \texttt{extract\_min()}, каждый из которых вытаскивает нам минимум из кучи за $\O(\log n)$, соответственно, алгоритм будет работать за $k\log n$
        \item Вот тут попробуем выкинуть все уровни кучи, которые находятся глубже $k$-ого, тогда для такой кучи \texttt{extract\_min()} будет отрабатывать $\O(\log k)$. Применим опять же $k$-раз \texttt{extract\_min()}, получим ответ за $\O(k\log k)$.
        
        \begin{remark}
            Так как \normalfont{\texttt{extract\_min()}} включает в себя процедуру просеивания вниз, которая работает за $\O(h)$, где $h$ --- глубина кучи, то с новой кучей, обрезанной по $k$-ый уровень, глубина, собственно говоря и есть $k$, отсюда и получается $\O\log k$ для обработки одного минимума.
        \end{remark}
    \end{enumerate}
\end{solution}
